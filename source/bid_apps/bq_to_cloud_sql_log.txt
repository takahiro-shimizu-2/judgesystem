● これが問題の核心です！ Cloud SQLがPrivate IPのみの場合、Cloud
  Runから接続する方法が変わります。

方法1: VPC Serverless Access Connector経由（推奨）

  Cloud SQLがPrivate IPのみの場合、Cloud RunをVPCに接続する必要があります。

  手順:

  1. Serverless VPC Access Connectorを作成
  gcloud compute networks vpc-access connectors create cloudrun-connector \
    --region=asia-northeast1 \
    --network=default \
    --range=10.8.0.0/28

  2. Cloud RunをVPC Connectorに接続
  デプロイスクリプトに以下を追加：
  --vpc-connector=cloudrun-connector \
  --vpc-egress=private-ranges-only

  3. 環境変数をPrivate IPに変更
  PGHOST=<Cloud SQLのPrivate IP>  # 例: 10.x.x.x

  方法2: Cloud SQLにパブリックIPを有効化

  セキュリティは下がりますが、最も簡単です。Cloud
  SQLでパブリックIPを有効化すれば、現在の設定のまま動作します。

  VPC Connector経由が推奨ですが、設定がやや複雑です。まずはCloud SQLのPrivate

 PostgreSQL側 調整コマンド例（Cloud SQL for PostgreSQL）
                                                                                     
  1. テーブルをUNLOGGED化（フルロード時のみ）                                        
                                                                                     
     psql "host=${HOST} dbname=${DB} user=${USER}" <<'SQL'                           
     DROP TABLE IF EXISTS backend_announcements CASCADE;                             
     CREATE UNLOGGED TABLE backend_announcements (                                   
         id TEXT PRIMARY KEY,
         no BIGINT,                                                                  
         orderer_id TEXT,                                                            
         title TEXT,                                                                 
         organization TEXT,                                                          
         category TEXT,                                                              
         bid_type TEXT,                                                              
         work_location TEXT,                                                         
         department_json JSONB,                                                      
         publish_date TEXT,                                                          
         explanation_start_date TEXT,                                                
         explanation_end_date TEXT,                                                  
         application_start_date TEXT,                                                
         application_end_date TEXT,                                                  
         bid_start_date TEXT,                                                        
         bid_end_date TEXT,                                                          
         deadline TEXT,                                                              
         estimated_amount_min NUMERIC,                                               
         estimated_amount_max NUMERIC,                                               
         status TEXT,                                                                
         actual_amount NUMERIC,                                                      
         winning_company_id TEXT,                                                    
         winning_company_name TEXT,                                                  
         competing_companies_json JSONB,                                             
         documents_json JSONB                                                        
     );                                                                              
     SQL                                                                             
  2. インポート前に不要なインデックス・トリガーを無効化                              
     (既に存在する場合)                                                              
                                                                                     
     -- 例：トリガー停止                                                             
     ALTER TABLE backend_announcements DISABLE TRIGGER ALL;                          
     -- 例：二次インデックス削除（後で作り直す）                                     
     DROP INDEX IF EXISTS idx_backend_announcements_category;                        
  3. Cloud SQL Import完了後のログ化とインデックス再作成                              
                                                                                     
     psql "host=${HOST} dbname=${DB} user=${USER}" <<'SQL'                           
     ALTER TABLE backend_announcements SET LOGGED;                                   
                                                                                     
     ALTER TABLE backend_announcements ENABLE TRIGGER ALL;                           
                                                                                     
     CREATE INDEX idx_backend_announcements_category                                 
       ON backend_announcements (category);                                          
     CREATE INDEX idx_backend_announcements_orderer                                  
       ON backend_announcements (orderer_id);                                        
     SQL                                                                             
  4. 統計更新
                                                                                     
     psql "host=${HOST} dbname=${DB} user=${USER}" -c \                              
       "VACUUM ANALYZE backend_announcements;"
  5. 差分ロード用（UPSERT）の準備                                                    
                                                                                     
     CREATE TABLE backend_announcements_staging (LIKE backend_announcements INCLUDING
  ALL);                                                                              
                                                                                     
     差分CSVを backend_announcements_staging に gcloud sql import csv で流し込み、完 
     了後：

     INSERT INTO backend_announcements AS target                                     
     SELECT * FROM backend_announcements_staging                                     
     ON CONFLICT (id) DO UPDATE SET                                                  
       no = EXCLUDED.no,                                                             
       orderer_id = EXCLUDED.orderer_id,                                             
       title = EXCLUDED.title,                                                       
       organization = EXCLUDED.organization,                                         
       category = EXCLUDED.category,                                                 
       bid_type = EXCLUDED.bid_type,                                                 
       work_location = EXCLUDED.work_location,                                       
       department_json = EXCLUDED.department_json,                                   
       publish_date = EXCLUDED.publish_date,                                         
       explanation_start_date = EXCLUDED.explanation_start_date,                     
       explanation_end_date = EXCLUDED.explanation_end_date,                         
       application_start_date = EXCLUDED.application_start_date,                     
       application_end_date = EXCLUDED.application_end_date,                         
       bid_start_date = EXCLUDED.bid_start_date,                                     
       bid_end_date = EXCLUDED.bid_end_date,                                         
       deadline = EXCLUDED.deadline,                                                 
       estimated_amount_min = EXCLUDED.estimated_amount_min,                         
       estimated_amount_max = EXCLUDED.estimated_amount_max,                         
       status = EXCLUDED.status,                                                     
       actual_amount = EXCLUDED.actual_amount,                                       
       winning_company_id = EXCLUDED.winning_company_id,                             
       winning_company_name = EXCLUDED.winning_company_name,                         
       competing_companies_json = EXCLUDED.competing_companies_json,                 
       documents_json = EXCLUDED.documents_json;                                     
     TRUNCATE backend_announcements_staging;                                         
                                                                                     
  これらをインポート前後のSQLとして自動化（Cloud Composer/Airflow など）すれば、20億 
  行規模のロードでも安定します。
