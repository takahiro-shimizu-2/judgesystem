# 要件定義書

## 1. システム概要

### 1.1 目的
入札公告に対する企業の参加資格を判定・管理し、入札業務のワークフローを効率化するWebアプリケーション

### 1.2 対象ユーザー
- 入札担当者
- 営業担当者
- 管理者

### 1.3 システム範囲
- 入札可否判定結果の一覧表示・検索
- 判定結果の詳細表示
- 入札業務ワークフロー管理
- 発注者・企業情報の管理

---

## 2. 機能要件

### 2.1 一覧画面機能

#### 2.1.1 データグリッド表示
| 項目 | 説明 |
|------|------|
| 表示列 | No、ステータス、優先順位、公告名、企業名、拠点、発注機関、カテゴリ、申請書提出日、判定日 |
| ソート | 各列でソート可能 |
| ページネーション | 25/50/100件単位で表示 |

#### 2.1.2 カード形式レイアウト
| 項目 | 説明 |
|------|------|
| 左側 | DataGridを含むカード（borderRadius: 4px） |
| 右側 | 選択アイテムの詳細プレビューパネル |
| 統一デザイン | 全一覧ページで同じレイアウトを使用 |

#### 2.1.3 フィルタリング機能
| 機能 | 説明 |
|------|------|
| 一括フィルター | ステータス、優先順位、工事カテゴリ、発注機関を一括選択 |
| 列フィルター | 各列に対して演算子（含む、等しい、始まる、終わる、空、空でない）を指定 |
| 日本語検索 | ひらがな/カタカナ/全角半角を同一視した検索 |
| 状態保存 | フィルター状態をlocalStorageに自動保存 |

#### 2.1.4 判定結果ステータス
| ステータス | 色 | 条件 |
|----------|-----|------|
| 参加可能 | 緑 | すべての要件を満たしている |
| 特殊条件 | オレンジ | 一部の特殊要件のみ未達 |
| 参加不可 | 赤 | 必須要件が未達 |

#### 2.1.5 作業ステータス
| ステータス | 色 | 説明 |
|----------|-----|------|
| 未着手 | グレー | 作業開始前 |
| 着手中 | 青 | 作業進行中 |
| 完了 | 緑 | 作業完了 |

#### 2.1.6 企業優先順位
| 優先度 | 表示 | 色 |
|--------|------|-----|
| 1 | ★★★ | 赤 |
| 2 | ★★☆ | オレンジ |
| 3 | ★★ | 黄 |
| 4 | ★☆ | 緑 |
| 5 | ★ | グレー |

### 2.2 詳細画面機能

#### 2.2.1 ヘッダー情報
- 公告タイトル
- ステータスバッジ
- 優先度バッジ
- 締切日カウントダウン
- 見積予想金額

#### 2.2.2 3カラムレイアウト

**左サイドバー（入札情報）**
| セクション | 内容 |
|----------|------|
| 資料リンク | 入札公告PDF、入札説明書、仕様書へのリンク |
| 概要情報 | 工事カテゴリ、工事場所 |
| スケジュール | 公告掲載日、説明書交付期間、申請書提出日、入札書提出期間 |
| 開札情報 | 開札結果（開札後に表示） |

**中央（ワークフロー）**
| ステップ | 内容 |
|---------|------|
| 判定結果 | 要件判定の詳細、達成率、各要件の詳細 |
| 発注者 | 確認事項、架電記録、事前提出資料 |
| 協力会社 | 候補リスト、資料管理 |
| 依頼 | 見積書、入札書、結果報告 |
| 落札情報 | 落札結果、参加企業一覧 |

**右サイドバー（連絡先情報）**
| セクション | 内容 |
|----------|------|
| 発注者情報 | 発注機関、担当者名、電話番号、住所、FAX、メール |
| 企業情報 | 企業名、支店名、住所、電話番号 |

#### 2.2.3 UI/UX要件
- 各セクションは開閉可能（アコーディオン）
- ワークフローステッパーでの進捗表示
- モバイル対応（レスポンシブデザイン）
- 統一borderRadius: 4px

## 3. 非機能要件

### 3.1 パフォーマンス
| 項目 | 要件 |
|------|------|
| 初期表示 | 3秒以内 |
| フィルター応答 | 500ms以内 |
| ページ遷移 | 1秒以内 |

### 3.2 ブラウザ対応
- Chrome（最新版）
- Firefox（最新版）
- Safari（最新版）
- Edge（最新版）

### 3.3 セキュリティ
- XSS対策（React標準のエスケープ処理）
- 外部リンクにrel="noopener noreferrer"設定

### 3.4 アクセシビリティ
- キーボード操作対応
- スクリーンリーダー対応（aria属性）
- カラーコントラスト比4.5:1以上

## 4. データ要件

### 4.1 入札判定データ
```typescript
interface BidEvaluation {
  id: string;
  evaluationNo: string;      // 判定連番
  announcement: Announcement; // 公告情報
  company: Company;          // 企業情報
  branch: Branch;            // 拠点情報
  requirements: Requirement[]; // 要件判定
  status: EvaluationStatus;  // 判定結果
  workStatus: WorkStatus;    // 作業ステータス
  evaluatedAt: string;       // 判定日時
}
```

### 4.2 公告情報
```typescript
interface Announcement {
  id: string;
  title: string;              // 工事名
  category: string;           // 工事カテゴリ
  organization: string;       // 発注機関
  workLocation: string;       // 工事場所
  department: Department;     // 担当部局
  publishDate: string;        // 公告掲載日
  deadline: string;           // 申請書提出日
  estimatedAmountMin?: number; // 見積予想金額下限
  estimatedAmountMax?: number; // 見積予想金額上限
}
```

### 4.3 企業情報
```typescript
interface Company {
  id: string;
  name: string;      // 企業名
  address: string;   // 住所
  grade: string;     // 等級
  priority: number;  // 優先順位(1-5)
}
```
